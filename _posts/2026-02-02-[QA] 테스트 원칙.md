# 🧪 Myers 핵심 원칙 (CSTS 기준)

> CSTS 교재 기준, Glenford J. Myers의 소프트웨어 테스트 핵심 요점 정리

---

## 1️⃣ 테스트 독립성 (Test Independence)

- 테스트는 **반드시 프로그램을 개발한 프로그래머나 팀과는 무관한 그룹**이 수행해야 한다.  
- 독립성을 확보해야 **객관적인 결함 발견**이 가능.

---

## 2️⃣ 결함 존재 가정 (Assume Defects Exist)

- 결함이 발견되지 않을 것이라는 가정으로 테스트 계획을 세워서는 안 된다.  
- 항상 **결함이 존재할 수 있다**는 전제 하에 테스트 설계 필요.

---

## 3️⃣ 정상/비정상 경우 모두 테스트 (Test Both Expected and Unexpected)

- **타당한 경우뿐 아니라 예상하지 못한 경우**에 대해서도 테스트 수행.  
- 예외 상황, 경계값, 잘못된 입력 등 다양한 시나리오 검증 필수.

---

## 4️⃣ 결함 집중 (Defect Clustering)

- 소프트웨어의 일부 모듈/기능에 **결함이 집중**되는 경향이 있음.  
- 결함이 많이 발견된 부분은 **추가 테스트 집중**.

---

## 5️⃣ 테스트 케이스 체계적 관리 (Systematic Test Case Management)

- 테스트 케이스를 **체계적으로 작성, 관리, 재사용**해야 함.  
- 추적성, 검증 가능성, 품질 관리를 위해 필수.

---

## 6️⃣ 테스트 결과 철저 검증 (Thorough Result Verification)

- 각각의 테스트 결과를 **철저히 점검**해야 함.  
- 단순 실행 기록에 그치지 않고, 결과 확인 및 분석까지 수행.

---

## 🧠 CSTS 기준 핵심 요약

| 원칙 | 의미 |
|------|------|
| 테스트 독립성 | 개발자와 무관한 그룹이 테스트 수행 |
| 결함 존재 가정 | 항상 결함 존재를 전제로 테스트 설계 |
| 정상/비정상 테스트 | 예상된 경우뿐 아니라 예외/비정상 상황 검증 |
| 결함 집중 | 일부 모듈/기능에 결함 집중 → 집중 테스트 |
| 테스트 케이스 관리 | 체계적 작성, 관리, 재사용 |
| 결과 철저 검증 | 테스트 결과를 단순 기록이 아닌 철저히 점검 |

---

> CSTS 기준으로 Myers 핵심 원칙을 따르면, **객관성 확보, 결함 발견, 체계적 관리**를 통해 제한된 시간과 자원 속에서도 **효율적이고 효과적인 QA 수행**이 가능하다.
