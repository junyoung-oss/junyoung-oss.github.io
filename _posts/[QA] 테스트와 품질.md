# 🏆 테스트와 품질

> 테스트는 품질을 보장하는 활동이 아니라, **품질 확보를 위한 수단**이다.  
> QA의 목적은 결함 발견뿐만 아니라, **품질 향상과 위험 최소화**에 있다.

---

## 📌 테스트와 품질의 관계

- **테스트(Test)**  
  - 소프트웨어의 결함을 발견하고, 기능이 요구사항을 만족하는지 검증  
  - 품질 향상의 **도구/수단**  

- **품질(Quality)**  
  - 요구사항 만족도, 안정성, 성능, 유지보수 용이성 등 소프트웨어 속성 전반  
  - 테스트만으로 보장 불가  
  - **조기 예방 + 지속적 개선** 필요

---

## 🧩 기능 테스트 vs 비기능 테스트

| 구분 | 목적 | 테스트 대상 |
|------|------|------------|
| **기능 테스트** | 요구사항대로 기능이 동작하는지 검증 | 로그인, 결제, 검색 등 기능 동작 |
| **비기능 테스트** | 소프트웨어 품질 속성 만족 여부 검증 | 성능, 안정성, 보안, 사용성 등 ISO 25010 품질 특성 기반 |

> 즉, 비기능 테스트는 **품질 요구사항 테스트**라고 이해하면 된다.

---

## 🔹 ISO/IEC 25010 소프트웨어 품질 특성

| 특성 | 설명 | 관련 비기능 테스트 예시 |
|------|------|----------------|
| **기능적 적합성** | 요구사항 충족, 올바른 결과 제공 여부 | 기능 테스트와 일부 중첩 |
| **성능 효율성** | 응답시간, 처리량, 자원 활용 수준 | 부하 테스트, 스트레스 테스트 |
| **호환성** | 다른 시스템, 플랫폼과 상호운용성 | OS/브라우저 호환성 테스트 |
| **사용성** | 학습 용이성, UI/UX 만족도 | UX 테스트, 접근성 테스트 |
| **신뢰성** | 오류 발생 빈도, 복구 능력, 안정성 | 장애 복구 테스트, 안정성 테스트 |
| **보안** | 기밀성, 무결성, 권한 관리 | 침투 테스트, 인증/권한 테스트 |
| **유지보수성** | 코드 수정 용이성, 모듈화 | 코드 구조 분석, 리팩토링 검증 |
| **이식성** | 환경 이동성, 설치 용이성 | 설치 테스트, 환경 이동 테스트 |

---

### 💡 QA 관점 연결

- ISO 25010 특성을 기준으로 비기능 테스트를 나누면 **누락 없이 체계적 검증** 가능  
- 기능 테스트는 기능 요구사항 검증,  
  비기능 테스트는 품질 요구사항 검증으로 이해  

---

## 🧩 QA 관점 핵심

1. **조기 품질 확보**  
   - 요구사항 단계부터 검토, 설계 리뷰, 정적 분석 등 포함
2. **위험 기반 테스트**  
   - 모든 케이스 테스트 불가 → 우선순위와 영향도 기반 검증
3. **결함 예방과 발견 병행**  
   - 테스트는 발견뿐 아니라 재발 방지와 프로세스 개선에도 사용
4. **테스트 품질 = 산출물 품질**  
   - 잘 정의된 테스트 케이스와 환경은 소프트웨어 품질과 직결
5. **측정과 피드백**  
   - 결함 통계, 테스트 커버리지, 리스크 분석 → 품질 개선 피드백

---

## 🔍 요약

| 항목 | 의미 |
|------|------|
| 테스트 | 결함 발견, 요구사항 검증 |
| 품질 | 요구사항 만족도, 안정성, 성능 등 소프트웨어 속성 |
| 기능 vs 비기능 | 기능 = 기능 요구사항 검증, 비기능 = 품질 요구사항 검증 |
| QA 역할 | 조기 검토, 위험 기반 테스트, 예방적 접근, 피드백 통한 지속 개선 |

---

> 핵심 메시지: 테스트는 **품질을 증명하는 도구**가 아니라,  
> **품질을 높이고 유지하기 위한 활동**이다.  
> ISO 25010 품질 특성을 기준으로 비기능 테스트를 체계화하면, 제한된 시간과 자원 속에서도 **효율적이고 신뢰성 높은 QA 수행**이 가능하다.
