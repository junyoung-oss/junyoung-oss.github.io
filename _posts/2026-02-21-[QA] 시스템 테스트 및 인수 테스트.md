# 📘 시스템 테스트 및 인수 테스트

---

# 📌 개요

시스템 테스트와 인수 테스트는 **개발 완료 후 전체 시스템**을 대상으로 수행되는 테스트이다.  
- 시스템 테스트: 소프트웨어 내부와 외부를 포함한 전체 동작 검증  
- 인수 테스트: 사용자 요구사항 충족 여부 확인

---

# 📌 시스템 테스트 (System Test)

## ✔ 개념
- 전체 시스템이 설계·요구사항을 충족하는지 검증

## ✔ 특징
- 단위/통합 테스트 완료 후 수행
- 기능 테스트 + 비기능 테스트 포함
- 전체 환경에서 실제 운영 조건과 유사하게 수행

## ✔ 목적
- 전체 시스템 동작 검증
- 시스템 상호작용 문제 및 성능/보안 등 비기능 검증
- 결함 발견과 배포 전 품질 확보

---

# 📌 인수 테스트 (Acceptance Test)

## ✔ 개념
- 사용자 관점에서 요구사항 충족 여부 확인  
- 실제 사용 환경과 유사한 조건에서 수행

## ✔ 특징
- 고객, 최종 사용자 참여 가능
- 기능/비기능 모두 검증 가능
- 배포 전 최종 승인 단계

## ✔ 목적
- 요구사항 기반 검증
- 사용자가 수용 가능한 품질 확인
- 배포 승인 여부 결정

---

# 📌 시스템 테스트 vs 인수 테스트 비교

| 구분 | 시스템 테스트 | 인수 테스트 |
|------|---------------|-------------|
| 대상 | 전체 시스템 | 전체 시스템 (사용자 관점) |
| 수행자 | QA 팀 | 고객/사용자, QA 팀 |
| 목적 | 설계·요구사항 충족 검증 | 요구사항 만족 + 배포 승인 |
| 범위 | 기능 + 비기능 | 주로 요구사항 기반 |
| 시점 | 통합 테스트 완료 후 | 시스템 테스트 완료 후 |

---

# 🧠 핵심 정리

- **시스템 테스트** = 전체 시스템 기능 + 비기능 검증, QA 중심  
- **인수 테스트** = 사용자 요구사항 만족 여부 확인, 배포 승인 판단  
- 두 단계 모두 소프트웨어 **완성도 확인과 품질 보증**에 필수

---

> 시스템 테스트와 인수 테스트는  
> 단위/통합 테스트 이후 **최종 품질 확보 단계**로,  
> 사용자 관점과 시스템 관점에서 모두 검증해야 한다.
