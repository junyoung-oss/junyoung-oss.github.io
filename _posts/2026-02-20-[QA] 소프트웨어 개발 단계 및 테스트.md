# 📘 소프트웨어 개발 단계 및 테스트

---

# 📌 개요

소프트웨어 개발 단계마다 수행되는 테스트의 역할과 유형이 달라진다.  
테스트는 단순히 결함을 찾는 활동이 아니라,  
개발 과정과 목표에 맞춰 전략적으로 수행되어야 한다.

---

# 📌 소프트웨어 테스트의 분류

테스트는 다양한 기준으로 분류할 수 있다.

---

## 1️⃣ 테스트 레벨별 분류

- **단위 테스트 (Unit Test)**  
  - 최소 단위(함수, 모듈) 검증
- **통합 테스트 (Integration Test)**  
  - 모듈 간 인터페이스 검증
- **시스템 테스트 (System Test)**  
  - 전체 시스템 검증
- **인수 테스트 (Acceptance Test)**  
  - 사용자 요구사항 충족 여부 확인

> 개발 단계에 맞춰 범위와 목표가 달라진다.

---

## 2️⃣ 테스트 설계 기법에 따른 분류

- **명세 기반 테스트 (Specification-based / Black-box)**  
  - 요구사항을 기반으로 케이스 설계
- **구조 기반 테스트 (Structure-based / White-box)**  
  - 코드 구조와 내부 로직 중심 설계
- **경험 기반 테스트 (Experience-based)**  
  - 테스터 경험과 과거 결함 기반 설계

> 설계 기법은 테스트 케이스를 어떻게 도출할지 결정

---

## 3️⃣ 테스트 유형별 분류 (기능/비기능)

- **기능 테스트 (Functional Test)**  
  - 요구사항 기능 검증
- **비기능 테스트 (Non-functional Test)**  
  - 성능, 보안, 사용성 등 품질 속성 검증

> “무엇을 검증할 것인가”라는 관점에서 구분

---

# 🧠 핵심 정리

- **개발 단계별** → 테스트 범위와 목적 결정  
- **설계 기법별** → 테스트 케이스 도출 방식  
- **유형별** → 기능/비기능 관점에 따른 검증 기준

---

> 테스트는 단일 활동이 아니라,  
> 개발 단계와 목적, 설계 기법, 유형에 따라 체계적으로 수행되는 활동이다.
