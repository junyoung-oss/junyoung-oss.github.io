## 1. 사이트 저장소 생성

사이트 저장소를 생성할 때는 두 가지 방법이 있습니다.

### 1-1. Starter 사용 (권장)

- 업그레이드를 쉽게 하고 불필요한 파일을 격리
- 최소한의 설정으로 글 작성에 집중 가능

**방법:**
1. GitHub에 로그인 후 [**starter**][starter] 페이지로 이동
2. **Use this template** 클릭 → **Create a new repository** 선택
3. 새 저장소 이름을 `<username>.github.io`로 지정 (`username`은 GitHub 계정 소문자)

### 1-2. 테마 포크(Fork) 방식

- 기능 또는 UI를 수정할 계획이 있는 경우 적합
- Jekyll에 익숙하지 않으면 추천하지 않음

**방법:**
1. GitHub 로그인
2. [테마 저장소 포크](https://github.com/cotes2020/jekyll-theme-chirpy/fork)
3. 새 저장소 이름을 `<username>.github.io`로 지정

## 2. 개발 환경 설정

저장소를 생성한 후에는 개발 환경을 설정해야 합니다. 주요 방법은 두 가지가 있습니다.

### 2-1. Dev Containers 사용 (Windows 권장)

Dev Containers는 Docker를 사용하여 격리된 환경을 제공하며, 시스템 충돌을 방지하고 모든 의존성을 컨테이너 내에서 관리할 수 있습니다.

**설치 단계:**

1. Docker 설치  
   - Windows/macOS: [Docker Desktop][docker-desktop] 설치  
   - Linux: [Docker Engine][docker-engine] 설치
2. [VS Code][vscode]와 [Dev Containers 확장][dev-containers] 설치
3. 저장소 클론  
   - Docker Desktop: VS Code에서 컨테이너 볼륨에 저장소 클론 [방법][dc-clone-in-vol]  
   - Docker Engine: 로컬에 클론 후 컨테이너에서 열기 [방법][dc-open-in-container]
4. Dev Containers 설정 완료될 때까지 대기

### 네이티브 환경 설정 (Unix 계열 OS 권장)

Unix 계열 시스템에서는 성능 최적화를 위해 네이티브 환경으로 설정할 수 있습니다. 물론 Dev Containers를 대신 사용할 수도 있습니다.

**설치 단계:**

1. [Jekyll 설치 가이드](https://jekyllrb.com/docs/installation/)를 따라 Jekyll 설치, [Git](https://git-scm.com/) 설치 확인
2. 로컬 컴퓨터에 저장소 클론
3. 테마를 포크했다면 [Node.js][nodejs] 설치 후 저장소 루트에서 `bash tools/init.sh` 실행
4. 저장소 루트에서 `bundle` 명령어 실행하여 의존성 설치

## 사용법

### Jekyll 서버 실행

로컬에서 사이트를 실행하려면 다음 명령어 사용:

```terminal
$ bundle exec jekyll serve
```

> Dev Containers를 사용하는 경우, 반드시 **VS Code 터미널**에서 실행해야 합니다.

몇 초 후 로컬 서버 접속 가능: [http://127.0.0.1:4000](http://127.0.0.1:4000)

### 설정

`_config.yml` 파일에서 필요한 변수를 업데이트하세요. 일반적으로 수정하는 항목:

- `url` : 사이트 URL
- `avatar` : 아바타 이미지
- `timezone` : 시간대
- `lang` : 언어 설정

### 소셜 연락처 옵션

사이드바 하단에 표시되며, `_data/contact.yml` 파일에서 특정 연락처를 활성화하거나 비활성화할 수 있습니다.
소셜 연락처 옵션은 사이드바 하단에 표시됩니다. `_data/contact.yml` 파일에서 특정 연락처를 활성화하거나 비활성화할 수 있습니다.

### 스타일시트 커스터마이징

스타일시트를 커스터마이징하려면, 테마의 `assets/css/jekyll-theme-chirpy.scss` 파일을 Jekyll 사이트의 동일 경로에 복사하고, 파일 끝에 원하는 스타일을 추가하세요.

### 정적 자산(Custom Static Assets) 설정

정적 자산 설정은 버전 5.1.0부터 지원됩니다.  
정적 자산의 CDN은 `_data/origin/cors.yml` 파일에서 정의되어 있으며, 사이트가 배포되는 지역의 네트워크 환경에 맞게 일부를 교체할 수 있습니다.  

자체 호스팅을 원할 경우 [_chirpy-static-assets_](https://github.com/cotes2020/chirpy-static-assets#readme) 저장소를 참고하세요.

## 배포

배포 전에 `_config.yml` 파일에서 `url`이 올바르게 설정되었는지 확인하세요.  
[**프로젝트 사이트**](https://help.github.com/en/github/working-with-github-pages/about-github-pages#types-of-github-pages-sites) 용도로 커스텀 도메인을 사용하지 않거나, GitHub Pages 외의 웹 서버에서 base URL로 접속하려면 `baseurl`을 프로젝트 이름(`/project-name`)으로 설정해야 합니다.

이제 Jekyll 사이트를 배포할 방법 중 하나를 선택할 수 있습니다.

### GitHub Actions를 이용한 배포

준비 사항:

- GitHub Free 플랜 사용 시 사이트 저장소는 공개 상태 유지
- 저장소에 `Gemfile.lock` 파일이 커밋되어 있고, 로컬 환경이 Linux가 아닌 경우, lock 파일의 플랫폼 목록을 업데이트

  ```console
  $ bundle lock --add-platform x86_64-linux
  ```

다음으로, _Pages_ 서비스를 설정합니다:

1. GitHub에서 저장소로 이동합니다. _Settings_ 탭을 선택한 후 왼쪽 메뉴에서 _Pages_ 클릭.  
   **Source** 섹션(빌드 및 배포)에서 [**GitHub Actions**][pages-workflow-src]를 드롭다운 메뉴에서 선택합니다.  
   ![Build source](pages-source-light.png){: .light .border .normal w='375' h='140' }
   ![Build source](pages-source-dark.png){: .dark .normal w='375' h='140' }

2. GitHub에 커밋을 푸시하면 _Actions_ 워크플로우가 실행됩니다. 저장소의 _Actions_ 탭에서 _Build and Deploy_ 워크플로우가 실행되는 것을 확인할 수 있습니다. 빌드가 완료되고 성공하면 사이트가 자동으로 배포됩니다.

이제 GitHub에서 제공하는 URL로 사이트에 접속할 수 있습니다.

### 수동 빌드 및 배포 (자체 서버)

자체 서버를 사용하는 경우, 로컬에서 사이트를 빌드한 후 사이트 파일을 서버에 업로드해야 합니다.

프로젝트 소스 루트로 이동한 후, 다음 명령어로 사이트를 빌드합니다:

```console
$ JEKYLL_ENV=production bundle exec jekyll b
```

출력 경로를 별도로 지정하지 않았다면, 생성된 사이트 파일은 프로젝트 루트 디렉토리의 `_site`{: .filepath} 폴더에 저장됩니다. 이 파일들을 서버에 업로드하면 됩니다.
