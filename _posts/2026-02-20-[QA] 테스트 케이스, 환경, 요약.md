# 📘 테스트 케이스와 테스트 환경

> 테스트는 개별 용어의 암기가 아니라  
> 용어들 사이의 관계를 이해하는 것이 핵심이다.

---

# 📌 테스트 기본 용어 간 관계

테스트는 다음의 구조 안에서 이해할 수 있다.

테스트 대상 → 테스트 레벨 → 테스트 유형 → 테스트 설계 기법 → 테스트 케이스 → 테스트 절차 → 테스트 스크립트 → 테스트 환경

---

## 1️⃣ 테스트 대상 (Test Item)

- 테스트의 대상이 되는 시스템 또는 구성 요소
- 모듈, 기능, API, 화면 등

👉 **무엇을 테스트하는가**에 대한 정의

---

## 2️⃣ 테스트 레벨 (Test Level)

- 테스트를 수행하는 범위의 단계 구분
  - 단위 → 통합 → 시스템 → 인수 테스트

👉 테스트 대상을 **어느 범위까지 묶어서 검증할 것인가**

---

## 3️⃣ 테스트 유형 (Test Type)

- 기능, 성능, 보안, 회귀 등 테스트의 목적/관점

👉 동일한 테스트 대상이라도  
어떤 관점으로 검증하는지에 따라 유형이 달라진다.

---

## 4️⃣ 테스트 설계 기법 (Test Design Technique)

### 테스트 케이스를 도출하는 방법
- 명세 기반
- 구조 기반
- 경험 기반

👉 테스트 유형과 요구사항을 바탕으로  
구체적인 테스트 케이스를 만들어내는 단계

---

## 5️⃣ 테스트 케이스 (Test Case)

- 입력 조건과 기대 결과를 정의한 문서

👉 설계 기법을 통해 도출된 **구체적인 검증 항목**

---

## 6️⃣ 테스트 절차 (Test Procedure)

- 테스트 케이스를 실제 수행하기 위한 단계별 실행 방법

👉 테스트 케이스를 **어떻게 실행할 것인지** 정의

---

## 7️⃣ 테스트 스크립트 (Test Script)

- 테스트 절차를 자동화 도구가 해석하고 실행할 수 있도록 작성된 코드

👉 테스트 절차의 자동화 형태

---

## 8️⃣ 테스트 환경 (Test Environment)

- 테스트가 수행되는 물리적·논리적 조건

👉 모든 테스트 활동은 특정 환경에서 수행된다.

---

# 🧠 전체 관계 요약

- 테스트 대상은 “무엇을”
- 테스트 레벨은 “어느 범위까지”
- 테스트 유형은 “어떤 관점으로”
- 설계 기법은 “어떻게 도출하고”
- 테스트 케이스는 “무엇을 검증하며”
- 테스트 절차는 “어떻게 실행하고”
- 테스트 스크립트는 “자동으로 실행하며”
- 테스트 환경은 “어디에서 수행하는가”를 의미한다.

---

> 테스트는 단어의 나열이 아니라,  
> 단계적으로 연결된 구조 속에서 이해해야 한다.
